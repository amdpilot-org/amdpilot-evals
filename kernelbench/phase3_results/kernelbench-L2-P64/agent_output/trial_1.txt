  Applied Optimizations

  1. torch.set_float32_matmul_precision('high'): Enabled TF32 tensor cores for f
     t32 matrix multiplication on MI355X, significantly speeding up the GEMM ope
     ion in the Linear layer. This was the major optimization.
  2. Triton fused LogSumExp + activations kernel: Fused kernel that combines Log
     Exp reduction with LeakyReLU x2 and GELU x2 into a single Triton kernel, re
     ing memory bandwidth and kernel launch overhead. Used manual tanh implement
     on for ROCm compatibility (since tl.math.tanh is unavailable).

  Key Learnings

  • MI355X (gfx950, CDNA4) has wavefront size 64, not 32 like NVIDIA
  • tl.math.tanh and tl.libdevice.* are unavailable on ROCm Triton - must use ma
    al implementation: (exp(2x) - 1) / (exp(2x) + 1)
  • torch.compile wrapping models with custom Triton kernels causes compatibilit
    issues with the custom Triton version in this environment
  • The GEMM operation dominates runtime (~95%); elementwise fusion provides mod
    t but meaningful improvement
