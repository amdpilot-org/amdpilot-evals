• Summary

  I successfully optimized KernelBench Level 2 Problem 67 (Conv2d + GELU + Globa
  lAvgPool) with Triton on AMD MI355X:

  Final Results

  • Baseline (PyTorch): 3.38ms
  • Optimized (Triton): 2.13ms
  • Speedup: 1.59x
  • Score: 65.9

  Key Optimizations Applied

  1. Used tl.math.erf for exact GELU: The initial tanh-based GELU approximation
     led correctness checks (max_diff=0.000128 > tolerance=1e-4). Switching to t
     ath.erf matched PyTorch's exact formula.
  2. Fused bias addition into Triton kernel: The biggest win (-0.77ms, 23% of ru
     me). Modified Conv2d to use bias=False, stored bias separately, and added i
     nside the Triton kernel before GELU. This eliminated the separate bias add
     nel entirely.
  3. BLOCK_SIZE tuning: Used BLOCK_SIZE=4096 which provides 16 iterations per bl
     for the 65536 spatial elements (256×256), balancing register pressure and l
     overhead.

  Kernel Structure

  • One thread block per (batch, channel) pair: 128 × 64 = 8,192 blocks
  • Each block processes all 65,536 spatial elements in chunks of 4,096
  • Operations fused: bias add → GELU → global average pool

  Remaining Bottleneck

  The Conv2d operation (miopenSp3AsmConv) still dominates at ~73% of runtime (~1
  .56ms), but this cannot be optimized with Triton in this task scope as it requ
  ires custom CUDA/HIP kernels or vendor library tuning.
